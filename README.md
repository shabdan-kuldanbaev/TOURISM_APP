# Tourism App

This is a Next.js project for a tourism application, bootstrapped with `create-next-app`.

## Architecture Overview

This project follows the Feature-Sliced Design (FSD) methodology. FSD is an architectural methodology for frontend applications that helps to create a scalable and maintainable codebase.

## Directory Structure

```
app/                          # Next.js App Router
src/
  shared/
    ui/                       # Shared UI components (shadcn/ui exports)
    lib/                      # Utility functions
    config/                   # Application configuration
    types/                    # Shared TypeScript types
    api/                      # API client configurations
  entities/                   # Business entities
  features/                   # User features
  widgets/                    # Composite blocks
  pages/                      # Page compositions (if needed)
sanity/                       # Sanity CMS (generated by init command)
  schemas/                    # Content schemas
  lib/                        # Sanity utilities
```

## Development

⚠️ **NEVER run these commands during development:**
- `npm run dev`
- `npm run start`
- `next dev`
- `next start`

These commands will freeze the AI assistant. Only run linting and formatting commands after development is complete.

## Available Commands

- `npm run build`: Builds the application for production.
- `npm run lint`: Lints the codebase using Biome.
- `npm run lint:fix`: Lints and fixes the codebase using Biome.
- `npm run format`: Formats the codebase using Biome.

## Technology Stack

- **Language**: TypeScript
- **Framework**: Next.js (latest, App Router)
- **Linter/Formatter**: Biome
- **UI Kit**: shadcn/ui
- **CMS**: Sanity CMS
- **CSS Framework**: Tailwind CSS (latest)
- **Architecture**: Feature-Sliced Design (FSD)
- **Bundler**: Webpack
